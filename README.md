# College Telegram Bot System

Два Telegram-боти (студентський та адміністративний) працюють в **одному процесі Python** і використовують **спільну базу даних**: локально — SQLite (`college.db`), на Railway — PostgreSQL через змінну середовища `DATABASE_URL`.

Інтерфейс для користувачів студентського бота — **українською**; тексти винесені в `locales/uk.json`.

## Локальний запуск

1. Python **3.11+** (для PostgreSQL на Windows зручніше **3.11 або 3.12**, щоб ставились готові колеса `asyncpg`).
2. Клонуйте репозиторій і перейдіть у каталог `college_bot`:

   ```bash
   cd college_bot
   ```

3. Встановіть залежності:

   ```bash
   pip install -r requirements.txt
   ```

   Якщо `asyncpg` не збирається (наприклад, Python 3.13 на Windows), для **лише SQLite** можна тимчасово встановити все, крім `asyncpg`, і не задавати `DATABASE_URL`.

4. Створіть файл `.env` і заповніть токени та ідентифікатори:

   - `BOT_TOKEN` — токен студентського бота  
   - `ADMIN_BOT_TOKEN` — токен адмін-бота  
   - `ADMIN_IDS` — ID адміністраторів через кому  
   - `ADMIN_CHAT_ID` — чат (група/канал), куди адмін-бот надсилає нові звернення зі студентського бота  
   - `DATABASE_URL` — залиште порожнім для SQLite  

5. Запуск:

   ```bash
   python main.py
   ```

## Деплой на Railway (GitHub)

У репозиторії вже є `railway.toml` (команда старту `python main.py`) та `runtime.txt` (Python 3.12).

1. У [Railway](https://railway.app) створіть **New Project** → **Deploy from GitHub** → оберіть репозиторій (наприклад `Reveno/ofcct_bot-telegram`).
2. Додайте плагін **PostgreSQL** в **той самий проєкт**, що й бот. У сервісі бота: **Variables** → **Add Variable Reference** → сервіс **Postgres** → додайте **`DATABASE_PRIVATE_URL`** (якщо Railway показує її в списку), інакше **`DATABASE_URL`**. Код спочатку читає `DATABASE_PRIVATE_URL` — це зменшує проблеми з мережею всередині Railway.
3. У **Variables** сервісу з ботом додайте вручну (значення з вашого `.env`, **не** комітьте їх у Git):
   - `BOT_TOKEN`, `ADMIN_BOT_TOKEN`
   - **`ADMIN_IDS`** — обов’язково ваш **особистий** числовий Telegram user id (через [@userinfobot](https://t.me/userinfobot) з **особистого** акаунта). Без цього адмін-бот відповість «Доступ заборонено». Кілька адмінів: `123456789,987654321`.
   - `ADMIN_CHAT_ID` — чат для сповіщень (група або ваш id).
   - за потреби `SOCIAL_*` URL
4. Запуск визначено в `railway.toml`. Після деплою перевірте **Deploy Logs** — мають ініціалізуватися обидва боти.
5. Якщо збірка не бачить залежності: **Root Directory** у налаштуваннях сервісу має вказувати на корінь репозиторію (там, де `main.py` та `requirements.txt`).

### Помилка `Connect call failed` / `Errno 111` до PostgreSQL

- Переконайтеся, що **Postgres і бот у одному проєкті**, а не в двох окремих.
- У **Variables** сервісу бота має бути **reference** на змінні з плагіна Postgres, а не ручний копіпаст з іншого місця.
- Додайте reference на **`DATABASE_PRIVATE_URL`**, якщо є; інакше — **`DATABASE_URL`**.
- Якщо після цього залишаються помилки SSL (рідко): додайте змінну `PGSSLMODE` = `require` у сервісі бота.
- Після змін змінних **Redeploy** сервісу. У коді є **повторні спроби підключення** (до ~1 хв), щоб Postgres встиг піднятися після деплою.

## Завантаження розкладу (Excel)

1. У **адмін-боті** виконайте `/uploadschedule` або меню «Розклад» → «Завантажити Excel».
2. Надішліть файл `.xlsx` у форматі, описаному в `utils/schedule_parser.py`.
3. Підтвердіть заміну поточного розкладу в базі.

## Зміни розкладу на поточний тиждень

1. Команда `/addchange` або меню «Зміни розкладу» → «Додати зміну».
2. Оберіть групу, день, номер пари, тип зміни (скасування / заміна / додаткова пара).
3. За потреби введіть предмет, викладача, кабінет, примітку — підтвердіть.
4. Зміни прив’язані до **понеділка поточного тижня** (`week_start` у форматі ISO).

Перегляд: `/listchanges` або відповідна кнопка в меню. Очистити всі зміни тижня: `/clearchanges`.

### Завантаження Excel у групі

- Можна надіслати **файл .xlsx** з підписом `/uploadschedule` одним повідомленням (або спочатку `/uploadschedule`, потім файл у приваті з ботом).
- У групах з кількома ботами використовуйте `/uploadschedule@ЮзернеймАдмінБота`.
- Якщо бот «не бачить» файли в групі: у [@BotFather](https://t.me/BotFather) → ваш адмін-бот → **Bot Settings → Group Privacy** → **Turn off** (або надсилайте файли в **приватний** чат з адмін-ботом).

## Довідник команд

### Студентський бот

| Дія | Опис |
|-----|------|
| `/start` | Реєстрація в базі, головне меню |
| `/cancel` | Вихід з діалогу (розклад, зворотний зв’язок) |

Розділи відкриваються через **інлайн-кнопки** головного меню: розклад, FAQ, зворотний зв’язок, соцмережі, перездачі, підписка, новини.

### Адмін-бот

| Команда / меню | Опис |
|----------------|------|
| `/start` | Панель адміністратора |
| `/messages` | Непрочитані звернення |
| `/reply USER_ID текст` | Відповідь студенту напряму |
| `/broadcast [текст]` | Розсилка підписникам |
| `/uploadschedule` | Завантаження Excel з розкладом |
| `/listgroups` | Список груп у базі |
| `/deleteschedule` | Майстер видалення однієї пари з розкладу |
| `/addchange` | Додати зміну на поточний тиждень |
| `/listchanges` | Список змін тижня |
| `/deletechange ID` | Видалити зміну за id |
| `/clearchanges` | Очистити всі зміни поточного тижня |
| `/addretake` | Додати перездачу |
| `/listretakes` | Список перездач |
| `/deleteretake ID` | Видалити перездачу |
| `/stats` | Статистика |
| `/cancel` | Скасувати активний діалог |

Не-адміністратори отримують повідомлення про заборону доступу (текст із локалізації).

## Структура проєкту

- `main.py` — ініціалізація БД, два `Application`, паралельний polling  
- `config.py` — змінні середовища, `LESSON_TIMES`, посилання для соцмереж  
- `db.py` — SQLite (`aiosqlite`) / PostgreSQL (`asyncpg`)  
- `i18n.py`, `locales/uk.json` — рядки інтерфейсу  
- `handlers/` — логіка студентського бота  
- `admin_handlers/` — логіка адмін-бота  
- `utils/schedule_parser.py` — розбір Excel  

## Ліцензія та підтримка

Налаштуйте URL соцмереж у `.env` (`SOCIAL_*`). Токени та `.env` не комітьте у репозиторій.
